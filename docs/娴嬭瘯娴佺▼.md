测试前准备


准备 3 个账号：
管理员账号
学生或家长账号
老师账号
确认老师账号已在管理端审核通过入驻。
推荐测试顺序

老师入驻链路
老师登录小程序。
进入「首页 -> 老师入驻」。
如果未入驻，提交入驻申请。
管理端审核通过。
老师重新进入小程序，确认底部第二个 Tab 显示为「工作台」。
老师发布家教信息
老师进入底部「工作台」。
点击「发布家教信息」。
填写日期、科目、年级、课时费、预期课时、说明。
发布成功后进入「工作台 -> 我的家教信息」。
测试编辑、下架、上架。
删除功能先用测试数据测，不要删正式数据。
学生找老师预约
学生登录小程序。
进入底部「找老师」。
在「老师库」里按地区、年级、科目筛选老师。
点进老师详情。
发起预约。
填写日期、时段、上课地址、联系方式、留言。
提交后去「我的 -> 我的订单 -> 课程预约」。
确认能看到老师姓名、时段、地址、联系方式、留言。
学生从家教信息预约
学生进入「找老师 -> 家教信息」。
找到老师刚发布的家教信息。
点击进入预约。
确认预约页的科目和课时费来自这条家教信息。
提交预约。
老师进入「工作台 -> 我的家教信息」，确认预约次数增加。
老师进入「工作台 -> 预约管理」，确认能看到这条预约。
老师处理预约
老师进入「工作台 -> 预约管理」。
在「待处理」里查看预约。
确认能看到学生联系方式和留言。
测试一条点击「同意」。
测试另一条点击「拒绝」。
学生回到「我的订单」确认状态变为「已确认」或「已取消」。
学生取消预约
对「待确认」预约点击「取消预约」。
填写取消原因。
确认订单状态变为「已取消」，并显示取消原因。
对「已确认」预约点击「申请取消」。
填写取消原因。
老师进入「工作台 -> 预约管理 -> 取消申请」。
确认看到取消原因。
老师点击「同意取消」。
学生确认订单状态变为「已取消」。
学生评价
学生只对「已确认」预约点击「去评价」。
提交 1～5 分和评价内容。
到「我的 -> 家长评价」确认评价立即显示。
对同一预约再提交一次，应该提示已评价过。
对未确认/已取消预约提交评价，应该被拦截。
输入 <script> 或明显违规词，应该被拦截。
商品订单
学生进入「进步商品」。
购买普通商品。
到「我的 -> 我的订单 -> 商品订单」确认展示。
学习课程订单
学生从首页热门课程或「进步商品」里的学习课程进入详情。
购买学习课程。
到「我的 -> 我的订单 -> 学习课程」确认展示。
智能选课助手
学生进入「智能选课助手」。
选择地区、年级、科目并输入问题。
点击获取建议。
确认返回专业建议和推荐老师信息。
如果 AI 调用失败，确认页面有降级提示或仍能展示平台推荐。
重点观察

小程序底部老师账号第二个 Tab 应该是「工作台」，学生账号是「找老师」。
学生端页面内应该是「老师库 / 家教信息」，不再是「找老师 / 选择家教」。
预约信息不应该再只塞在地址里，老师端和订单页应分别展示时段、地址、联系方式、留言。
小程序「我的」页面不应再出现会员卡入口。
若出现 SQL 字段不存在，说明后端连的不是你刚执行 SQL 的 sunhao 数据库，或后端没重启。
