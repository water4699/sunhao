### B.1 已落地（代码侧，仍需你验收勾选）

- [x] **工程**：`Teacher` 实体与 `TeacherMapper` 支持 `areaId`/`gradeId`；列表查询可按地区、年级、科目、姓名筛选。

- [x] **小程序 · 预约**：`appointmentTeacher.vue` 调用 `POST /system/course`；`startDate` 仅 **日期**（`yyyy-MM-dd`），**上课时段 + 详细地址**写入 **`address`**（格式：`时段：HH:mm；用户输入地址`）；未登录时后端拒绝；成功后跳转「我的订单」Tab。

### B.4 小程序与管理端同步检查（代码快照，2026-04）

以下指 **同一后端接口下**，uni-app（`uniapp/juxing-ruoyi`）与 `ruoyi-ui/src/views/my/*` 字段、数据流是否对齐。

| 业务 | 小程序 | 管理端 | 结论 |
|------|--------|--------|------|
| **教师** | `GET/POST /system/teacher`，入驻提交含 `areaId`、`gradeId`、`subjectId`；列表带 `status=1` | `my/teacher/index` 含 **地区/年级** 查询、表格展示（名称映射）、编辑表单下拉 | **已同步**（以库表与字典数据为准） |
| **课程/预约** | `POST /system/course`（**重复预约后端拒绝**）、`GET /system/course/list?studentId=`；`startDate`+`address` 见 B.1 | `my/course/index` 已加 **上课地址**；**确认课程 `status=1`** 后家长才可评价 | **已同步**（评价门槛依赖后台改预约状态） |
| **商品** | `GET /system/product/list`；**下单** `POST /system/order/app/product`（mock） | `my/product` 维护商品；`my/order` 可看 **`order` 表** | **已同步**（小程序订单 Tab 拉 `GET /system/order/app/list`） |
| **评价** | **提交** `POST /system/review/app/submit`；**公开展示** `GET /system/review/app/list`（仅 `status=1`） | `my/review`：**待审核 `0` → 通过 `1`** 后小程序评论区可见 | **已同步**（提交走 app 接口；后台列表仍可用原 `list`） |
| **订单 order 表** | 「我的订单」**商品订单** Tab + 确认页 **模拟支付** | `my/order` 维护 `order`；**约定 `course_id` 存 `productId`** | **已对接**（非真实支付，论文需说明） |
| **会员卡** | 入口 **toast**；子页为 **占位**（不调用接口） | `my` 下会员卡配置 | **已停用占位**；接真实业务时再与后台对齐 |

**结论**：教师、课程预约（含重复校验）、商品与 **order**、评价 **提交+公开展示链路** 已与后端对齐；剩余主要是 **注册/角色/答辩账号**、**阶段 C AI**、**测试与文档**；会员卡 **暂不开放**。
