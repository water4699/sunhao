### B.4 小程序与管理端同步检查（代码快照，2026-04）

以下指 **同一后端接口下**，uni-app（`uniapp/juxing-ruoyi`）与 `ruoyi-ui/src/views/my/*` 字段、数据流是否对齐。

| 业务 | 小程序 | 管理端 | 结论 |
|------|--------|--------|------|
| **教师** | `GET/POST /system/teacher`，入驻提交含 `areaId`、`gradeId`、`subjectId`；列表带 `status=1` | `my/teacher/index` 含 **地区/年级** 查询、表格展示（名称映射）、编辑表单下拉 | **已同步**（以库表与字典数据为准） |
| **课程/预约** | `POST /system/course`、`GET /system/course/list?studentId=`；`startDate`+`address` 约定见 B.1 | `my/course/index` 已加 **上课地址** 列与表单；列表可看学生/教师 ID | **已同步**（管理端表单字段仍较简，复杂录入可只依赖小程序） |
| **商品** | `GET /system/product/list`，前端按 `type` 分类 | `my/product` 等维护商品 | **已同步**（接口一致；分类依赖后台 `type` 字段与小程序写死分类名一致） |
| **评价** | `GET /system/review/list`（已去掉占位假数据与错误下拉逻辑） | `my/review` 或系统评价菜单 | **列表已同步**；小程序**无提交评价入口**的需另页对接 `POST /system/review`（待 B.2） |
| **订单 order 表** | 当前「我的订单」页展示的是 **课程预约（course）** | `my/order` 维护 `order` 表 | **未混用**：商品订单若要走 `ruoyi-order`，小程序需单独页面/接口（待 B.2） |
| **会员卡** | 入口 **toast**；子页为 **占位**（不调用接口） | `my` 下会员卡配置 | **已停用占位**；接真实业务时再与后台对齐 |

**结论**：**商品订单（order 表）**、评价 **提交** 仍为两端主要差距；教师地区/年级、课程预约、商品列表、评价 **列表** 已与后端及管理端对齐；会员卡 **暂不开放**。
