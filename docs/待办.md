## 阶段 B：与说明书一致的业务闭环（端到端可演示）

对照需求基线逐项自检：**注册登录、教师信息发布/审核、找老师、地区/年级/科目筛选、预约课程、订单、商品、会员卡、评价**。

- [ ] **注册与登录**：家长/教师角色注册、登录、token 刷新与退出；与若依权限模型一致（小程序端实际使用的角色标识与后端一致）。
- [ ] **教师侧**：信息发布、编辑；与管理端「审核」状态联动（待审/通过/驳回）；教师入驻流程无半截逻辑。
- [ ] **家长侧 · 找老师**：列表与筛选（地区、年级、科目）走后端接口；分页、空态、错误提示完整；删除页面内长期占位示例数据。
- [ ] **预约与课程**：预约提交、状态展示、与订单/课程模块数据一致；异常场景（重复预约、时间冲突等）有提示或后端约束说明。
- [ ] **订单与商品**：下单、支付流程（若说明书要求真实支付则对接或说明模拟策略）、订单列表与详情与后端一致。
- [ ] **会员卡**：购买/绑定/使用规则与后台配置一致；边界情况有提示。
- [ ] **评价**：提交评价、展示、与管理端审核或展示策略一致（按你论文设定）。
- [ ] **管理端（PC）**：对以上业务在 `ruoyi-ui` 中可配置、可查询、可演示；关键菜单与权限分配给答辩账号。
