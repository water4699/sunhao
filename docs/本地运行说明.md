# 本地运行说明（聚星 / RuoYi 多端）

本文档对应阶段 A 中的「环境配置」与「本地重新构建」说明，便于在本机启动后端、管理端与 uni-app 小程序/H5。

---

## 1. 环境要求（建议版本）

| 组件 | 建议版本 | 说明 |
|------|----------|------|
| JDK | **8**（1.8） | 根 `pom.xml` 中 `<java.version>1.8</java.version>` |
| Maven | 3.6+ | 用于编译 `master/RuoYi-Vue-master` |
| MySQL | 5.7 / 8.0 | 与若依官方要求一致；导入项目 SQL |
| Redis | 5.x / 6.x | 若依登录会话、缓存等依赖 |
| Node.js | 14.x～18.x LTS | 用于 `ruoyi-ui` 前端工程 |
| HBuilderX 或 CLI | 按 DCloud 文档 | 编译/运行 `uniapp/juxing-ruoyi` |

具体数据库名、账号密码以 `ruoyi-admin/src/main/resources/application-druid.yml`（或你本地使用的 profile）为准，**勿将含密码的配置提交到公开仓库**。

---

## 2. 后端（Spring Boot）

路径：`master/RuoYi-Vue-master`

1. 创建数据库并执行若依提供的 SQL 脚本（通常在 `sql/` 目录，以仓库内实际文件为准）。
2. 修改数据源与 Redis 配置（`application.yml` / `application-druid.yml`）。
3. 编译（在项目根目录）：

```bash
cd master/RuoYi-Vue-master
mvn -DskipTests compile
```

打包与运行入口模块一般为 `ruoyi-admin`（以官方文档为准）：

```bash
mvn -pl ruoyi-admin -am package -DskipTests
java -jar ruoyi-admin/target/ruoyi-admin.jar
```

4. 默认后端端口多为 **8080**（以 `application.yml` 中 `server.port` 为准）。

---

## 3. 管理端（Vue / `ruoyi-ui`）

路径：`master/RuoYi-Vue-master/ruoyi-ui`

- **开发环境代理**：`vue.config.js` 中 `baseUrl` 指向后端，例如 `http://localhost:8080`；`VUE_APP_BASE_API` 在 `.env.development` 中为 `/dev-api`，由 devServer 代理到上述后端。
- 安装依赖并启动（需自行安装 Node；本仓库 `.gitignore` 忽略 `node_modules`，克隆后需重新 `npm install`）：

```bash
cd master/RuoYi-Vue-master/ruoyi-ui
npm install
npm run dev
```

- 浏览器访问端口以控制台输出为准（`vue.config.js` 中常见为 **80**，若被占用请改端口或换机器端口）。

生产构建：`npm run build:prod`，产物在 `dist/`，由 Nginx 等静态托管并与后端反向代理配合（参见若依官方部署文档）。

## 4. 小程序 / H5（uni-app）

路径：`uniapp/juxing-ruoyi`

### 4.1 接口地址（`config.js`）

```javascript
// uniapp/juxing-ruoyi/config.js
baseUrl: 'http://localhost:8080'
```

- 真机调试小程序时，`localhost` 指向手机自身，**不可访问电脑后端**。请改为：
  - 本机局域网 IP（如 `http://192.168.x.x:8080`），或
  - 已部署的 HTTPS 域名（微信小程序要求配置**合法 request 域名**）。
- 注释中的 `https://vue.ruoyi.vip/prod-api` 为官方演示环境，仅作参考。

### 4.2 本地重新构建（产物可再生）

- `unpackage/` 为编译输出目录，**不应作为核心源码依赖**；已在仓库根 `.gitignore` 与 `uniapp/juxing-ruoyi/.gitignore` 中忽略。
- 使用 **HBuilderX**：打开 `uniapp/juxing-ruoyi`，选择运行到浏览器 / 微信开发者工具。
- 使用 **CLI**（若已配置）：按项目内 `package.json` 脚本执行（若有）；无统一脚本时以 HBuilderX 为准。

### 4.3 微信小程序

- `manifest.json` → `mp-weixin` 中配置小程序 AppID。
- 开发者工具中开启「不校验合法域名」仅用于开发；上架前须在公众平台配置服务器域名与业务域名。

### 4.4 H5

- `manifest.json` 中 H5 `devServer.port` 当前为 **9090**（以文件为准），与后端 8080 互不冲突。

## 5. 联调检查清单

- [ ] MySQL、Redis 已启动，后端日志无连接错误。
- [ ] 浏览器可访问管理端，且接口经代理返回正常。
- [ ] `config.js` 的 `baseUrl` 与后端实际地址一致（含端口）。
- [ ] 小程序真机/模拟器可访问后端（域名或 IP 可达、HTTPS/合法域名已配置）。
