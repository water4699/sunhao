# 聚星教育业务状态码说明

## 表边界

- `course`：学生/家长向老师发起的预约单，逻辑名称为 `course_booking`。
- `teacher_published_course`：老师发布的可接单家教信息，逻辑名称为 `teacher_tutor_post`。
- 商品订单中的学习课程：属于进步商品/订单链路，逻辑名称为 `learning_course_product`，不要和预约单混用。

当前不直接重命名物理表，原因是管理端代码、Mapper、历史 SQL 已经依赖旧表名；后续若要重命名，应单独做数据迁移和兼容视图。

## 老师入驻/教师审核状态

- `0`：待审核
- `1`：已通过
- `2`：已拒绝

## 预约单 `course.status`

- `0`：待老师确认
- `1`：老师已确认
- `2`：已拒绝或已取消
- `3`：学生已申请取消，等待老师确认

规则：
- 学生在 `0` 时取消，直接变为 `2`。
- 学生在 `1` 时取消，先变为 `3`。
- 老师对 `3` 的处理只允许同意取消，最终变为 `2`。
- 评价只允许在 `1` 状态提交，且同一预约只能评价一次。

## 老师家教信息 `teacher_published_course.status`

- `0`：上架中，学生端「家教信息」可见
- `1`：已下架，学生端不可见，老师端「我的家教信息」仍可管理
